# Tablero Comercial CEA v1.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply the 8 changes in the "ACTUALIZACIÓN TABLERO COMERCIAL CEA — v1.1" spec to the existing Streamlit dashboard without rebuilding anything that already works: a new Tab 6 (OKRs y Plan), an app-wide abbreviation glossary, real 2024-2025 market figures, an app-wide "nan"/"None" fix, a CRM→"Excel de Ventas" rename, real costos_fijos.json data, a discount-policy text fix, and a new "Precios y promociones" block atop Tab 5.

**Architecture:** No new subsystems. `data/okrs.json` follows the same static-JSON-versioned-in-repo pattern as the other 5 data files; `tabs/tab6_okrs.py` follows the same `render(con, hojas, es_demo)` contract as tabs 1-5; `src/ui.py` gains three small pure helpers (`GLOSARIO`, `definicion`, `fmt`) that the existing tabs call into, rather than each tab reinventing null-handling.

**Tech Stack:** Same as today — Streamlit, DuckDB, pandas, plotly, gspread. No new dependencies.

## Global Constraints

- UI language: Spanish only, no anglicisms (established pattern — see `src/constants.py` docstring).
- Data provenance marks `✅`/`📊`/`🎯` must be preserved on every figure, exactly as in the existing JSON files.
- "CRM" must never appear in anything the user sees in the running app (title text, captions, spinners, warnings) — it is renamed to "Excel de Ventas" everywhere user-visible. Internal variable/file names (`sheets.py`, `hoja`, `HOJAS`, etc.) are unchanged — this is a display-string rename only.
- No value that is `None`/`null`/`NaN` may ever reach the screen as the literal text "nan", "None", or a blank cell — it must render as "Por llenar" via the new `src/ui.py::fmt()` helper.
- The word "firma" (as in "requires Eduardo's signature") and the literal string "Eduardo García Alanis" must not appear anywhere in the running app. **Judgment call, flagged for your review:** the OKR source document legitimately uses "firmadas"/"firmado" in KR3.3 ("decisiones... firmadas o rechazadas") and KR4.3 ("convenio firmado") to describe real business facts (a decision being signed off, a contract being signed) — unrelated to the "pending Eduardo's signature" phrasing the spec is scrubbing. Those two KRs are transcribed verbatim from the source doc in Task 10. The only edits made for this constraint are: (a) the Tab 5 discount-policy warning text (Task 5 + Task 12), and (b) INI-E10's owner field, changed from the source doc's "Eduardo (firma) + Paola" to "Eduardo + Paola" (Task 10) since that parenthetical is pure role-annotation, not business content, and dropping it costs nothing.
- No test suite exists in this repo (`find` turned up no `tests/` directory) and this is a Streamlit UI project — introducing pytest scaffolding is out of scope (YAGNI, and the spec says "NO reconstruir lo que ya funciona"). Verification in this plan instead uses: `python -c "import json; json.load(...)"` for every JSON edit, `python -c "import ast; ast.parse(open(...).read())"` / direct module import for every Python edit, targeted `grep` checks for the checklist's banned strings, and one end-to-end `streamlit run app.py` demo-mode smoke test in the final task.
- Every task must leave the app runnable in demo mode (`streamlit run app.py` with no `.streamlit/secrets.toml` configured) — this is checklist item "Modo demo sigue funcionando sin credenciales".

```python
def definicion(clave: str) -> str:
    """Regresa el texto de glosario para `clave`, o cadena vacía si no existe."""
    return GLOSARIO.get(clave, "")


def nota_glosario(clave: str) -> None:
    """Imprime la definición de `clave` en itálicas/letra chica, si existe."""
    texto = GLOSARIO.get(clave)
    if texto:
        st.caption(f"_{texto}_")


def fmt(valor, sufijo: str = "") -> str:
    """Formatea `valor` para mostrar en UI; nunca deja pasar 'nan'/'None'/vacío.

    None, NaN (float o pandas) -> 'Por llenar'. Números -> con separador de
    miles. Cualquier otro valor -> str(valor) + sufijo.
    """
    if valor is None:
        return "Por llenar"
    if isinstance(valor, float) and math.isnan(valor):
        return "Por llenar"
    try:
        if pd.isna(valor):
            return "Por llenar"
    except (TypeError, ValueError):
        pass
    if isinstance(valor, (int, float)):
        return f"{valor:,.0f}{sufijo}"
    return f"{valor}{sufijo}"


def punto_score_okr(avance: float | None) -> str:
    """Semáforo de score de KR: ⚪ sin registro, 🟢≥0.7, 🟡0.3-0.69, 🔴<0.3."""
    if avance is None:
        return "⚪"
    color = color_score_okr(avance)
    if color == COLOR_VERDE:
        return "🟢"
    if color == COLOR_AMARILLO:
        return "🟡"
    return "🔴"
```

```python
import json
from pathlib import Path

import pandas as pd
import streamlit as st

from src import ui

DATA_DIR = Path(__file__).resolve().parent.parent / "data"


def _avance_desde_sheet(hojas: dict) -> dict[str, float]:
    """Lee la hoja OKR_Avance si existe y trae datos; regresa {kr_id: avance}."""
    df = hojas.get("OKR_Avance", pd.DataFrame())
    if df.empty or "kr_id" not in df.columns or "avance" not in df.columns:
        return {}
    out = {}
    for _, r in df.iterrows():
        try:
            out[r["kr_id"]] = float(r["avance"])
        except (TypeError, ValueError):
            continue
    return out


def _fila_kr(kr: dict, avance_override: dict[str, float]) -> None:
    avance = avance_override.get(kr["id"], kr["avance"])
    col_marca, col_texto, col_barra = st.columns([0.6, 4, 2])
    with col_marca:
        st.markdown(kr.get("marca", "") or "&nbsp;", unsafe_allow_html=True)
    with col_texto:
        st.markdown(f"**{kr['id']}** {kr['texto']}")
    with col_barra:
        if avance is None:
            st.progress(0)
            st.caption("⚪ Sin registro")
        else:
            st.progress(min(max(avance, 0.0), 1.0))
            st.caption(f"{ui.punto_score_okr(avance)} {avance:.0%}")


def render(con, hojas, es_demo: bool) -> None:
    ui.encabezado_tab("¿Qué apostamos este trimestre y cómo vamos?")

    with open(DATA_DIR / "okrs.json", encoding="utf-8") as f:
        okrs = json.load(f)

    avance_override = _avance_desde_sheet(hojas)
    if not avance_override:
        st.caption("📊 Modo estático — sin hoja `OKR_Avance` en el Excel de Ventas todavía; los KR muestran 'Sin registro'.")

    # ------------------------------------------------------------------
    # 1. Tarjeta Norte
    # ------------------------------------------------------------------

        f"""
        <div style="border-left: 8px solid #2E7D32; padding: 1rem 1.4rem;
                    background: rgba(46,125,50,0.08); border-radius: 8px; margin-bottom: 0.8rem;">
```
